Especificações

Perspetiva do Produto

Pretende-se desenvolver uma aplicação que venha colmatar a necessidade de várias pessoas, não só utilizadores dos transportes públicos mas também os gestores/administradores dos mesmos, que desejam obter diversas informações sobre os autocarros da cidade do Porto.

A aplicação OnTrack, ainda em desenvolvimento, permitirá, a qualquer utilizador, obter informações relativas à localização dos vários autocarros, bem como um conjunto de aspetos técnicos relativos a um determinado autocarro, como por exemplo a sua velocidade média, número de km’s percorridos, entre outros. Será também possível efetuar a filtragem de autocarros por parte de um utilizador, o que lhe permitirá definir um conjunto de atributos para escolher mais facilmente o autocarro ideal para satisfazer as suas necessidades de deslocamento dentro da cidade do Porto.

Assim, esta aplicação pretende trazer comodidade a qualquer pessoa que escolha desfrutar das suas funcionalidades.

Funcionalidades do Sistema

Este sistema tem como objetivo satisfazer as necessidades dos seus utilizadores finais, de forma a complementar sistemas já existentes, como o Porto.Bus, oferecendo informações em tempo real acerca dos autocarros da cidade do Porto, não só aos utilizadores destes transportes públicos como também aos gestores dos mesmos.

Após um levantamento de necessidades de diversos possíveis utilizadores e dos cenários de utilização destes, foram definidas as seguintes funcionalidades:


F01 Visualização em tempo real dos autocarros e das suas trajetórias na cidade do Porto.
F02 Visualização em tempo real das paragens de autocarros da cidade do Porto.
F03 Definição de um alarme para um determinado autocarro.
F04 Verificação dos autocarros dentro de uma determinada área de localização.
F05 Filtragem de autocarros, em função de um conjunto de atributos a definir pelo utilizador.
F06 Visualização do histórico de um determinado autocarro.
F07 Obtenção de dados estatísticos sobre os autocarros.

Interfaces de Software

Backend

Dataset A aplicação depende dos dados recolhidos por sensores nos autocarros do Porto. Contudo, para efeitos de simulação e teste foi usado um dataset que contém previamente dados retirados desses sensores, como o ID do autocarro, as coordenadas, a velocidade média, entre outros.
Data Storage Para o armazenamento de dados, a aplicação OnTrack faz uso de duas bases de dados. Para armazenar as informações de cada conta de utilizador é usada uma base de dados MySQL. Relativamente ao armazenamento das informações dos autocarros, a aplicação dispõe de uma base de dados InflluxBD que facilita a recolha e tratamento dos dados.
Web Framework
Data Stream

Frontend

Web Application A interface de utilizador da aplicação OnTrack está construída em React.

Requisitos de Usabilidade

USE01 O sistema deverá permitir visualizar, de forma imediata, a localização e trajetória dos autocarros.
USE02 O sistema deverá permitir visualizar, de forma imediata, a localização das paragens dos autocarros.
USE03 A filtragem dos autocarros, por parte do utilizador, deverá ser possível em menos de 1 minuto.
USE04/b> A seleção de uma paragem deve ser possível em menos de 30 segundos.
USE05 A seleção de um autocarro e apresentação dos seus dados, como por exemplo, a sua velocidade, deverá ser possível em menos de 30 segundos.
USE06 A seleção da área dos autocarros deverá ser possível em menos de 1 minuto.
USE07 Para todas as interações do utilizador com a aplicação, deverá ser possível reverter a funcionalidade ou até mesmo cancelar.
USE08 A interface do sistema deverá ser simples, intuitiva e direta.
USE09 O início de sessão deverá ser possível em menos de 30 segundos.
