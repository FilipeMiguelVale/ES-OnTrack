OnTrack

A aplicação OnTrack foi desenvolvida no ambito da unidade curricular Engenharia de Software, do ano 4 do Mestrado Integrado em Engenharia de Computadores e Telemática, no ano letivo 2020-2021.

Visão do sistema

O objetivo do OnTrack é proporcionar uma sistema simples e interativo, que serve não só os utilizadores dos autocarros do Porto como também o administrador/gestor do sistema.

Milestones

É possível aceder à distribuição de trabalho do grupo e por elemento aqui.

Milestone 0

(Week 08)

Primeiramente, a equipa definiu o tema e o desenvolvimento do conceito associado a este, elaborando a versão primordial do contrato a ser cumprido ao longo do desenvolvimento do projeto.

Milestone 1

(Week 10)

Após a escolha do tema e a definição do produto, a equipa começou a elaborar a documentação necessária, como a Visão, a Arquitetura e a especificação dos Requisitos associados ao sistema. Também se estabeleceu a estrutura do presente site de documentação, que irá ser atualizado ao longo do desenvolvimento do sistema.

Milestone 2

(Week 12)

De seguida, a equipa dedicou-se ao desenvolvimento do sistema, onde começou por criar um pipeline para o Jenkins de forma a permitir fazer Continuous Integration e Continuous Delivery, implementou a base de dados do sistema e o Kafka e criou a plataforma REACT para a interação com o utilizador.

Milestone 3

(Week 14)

De seguida, a equipa completou as seguintes tarefas: fornecer de dados, inserir dados, eventos baseado em kafka (definição de tópicos básicos e os fluxos do sistema), backoffice da web/REST (consumidor de dados que fornece uma interface REST para alimentar a aplicação web) e a aplicação web (como componente externo). Nesta fase, concluiu-se a implantação completa do projeto baseada em Jenkins e desenvolveram-se os testes de integração do sistema.

Milestone 4

(Week 15)

Numa penúltima fase, a equipa completou o pipeline DevOps compatível com Jenkins e a camada de monitoramento ELK.

Milestone 5

(Week 16)

Por fim, a equipa focou-se na verificação de todo o sistema, de modo a confirmar se o mesmo funcionava como o esperado.
